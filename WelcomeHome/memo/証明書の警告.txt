HTTPSサーバーを起動した後、ローカル環境でアプリにアクセスするには、以下のURLをブラウザのアドレスバーに入力します：

```
https://localhost:8080
```

ここで、`8080`はサーバーがリッスンしているポート番号です。サーバーの設定によっては、異なるポート番号を使用している場合がありますので、その場合は適切なポート番号を使用してください。

以下は、HTTPSサーバーの起動方法とアクセス手順の詳細です。

### 1. HTTPSサーバーの起動
`main.js`を起動してHTTPSサーバーを開始します。ターミナルで以下のコマンドを実行します。

```sh
node main.js
```

サーバーが正しく起動している場合、ターミナルには次のようなメッセージが表示されます：

```
Server is running on port 8080
```

### 2. ブラウザからアクセス
ブラウザのアドレスバーに以下のURLを入力します：

```
https://localhost:8080
```

このURLは、`main.js`ファイル内で設定したHTTPSサーバーがリッスンしているポート番号を基にしています。デフォルトではポート`8080`を使用していますが、環境変数`PORT`が設定されている場合はその値を使用します。

### 証明書の警告について
自己署名証明書を使用しているため、初めてアクセスするときにブラウザが「この接続は安全ではない」という警告を表示することがあります。これは正常な動作です。以下の手順で続行してください：

1. **Chrome**：
   - 「詳細設定」をクリックします。
   - 「localhost にアクセスする (安全ではありません)」リンクをクリックします。

2. **Firefox**：
   - 「詳細情報」をクリックします。
   - 「危険性を理解した上で続行」ボタンをクリックします。

### アクセス例
以下のスクリーンショットは、ブラウザのアドレスバーにURLを入力した例です。

![Browser Address Bar](https://example.com/path/to/image)

これで、HTTPSサーバーに接続して、カメラの映像を表示するアプリケーションにアクセスできるはずです。

### 注意点
- ブラウザのカメラアクセスを許可する必要があります。通常、初めてアクセスするときにブラウザがカメラアクセスの許可を求めるポップアップを表示します。
- サーバーの設定や証明書のパスが正しく設定されていることを確認してください。証明書と秘密鍵のパスが間違っている場合、サーバーは正しく起動しません。

これらの手順に従うことで、ローカル環境でHTTPSサーバーを使用してアプリケーションにアクセスできるようになります。
